Comment créer une API REST avec Node.js et Express

Un guide étape par étape pour construire une API REST simple avec Node.js, Express et MongoDB, incluant la gestion des routes, des données et des erreurs.

Introduction

Les API REST sont aujourd'hui au cœur de la majorité des applications web modernes. Elles permettent à différentes applications de communiquer entre elles de manière simple, rapide et standardisée.

Lors de mes projets en développement web, j'ai découvert l'importance des API pour connecter une interface utilisateur à une base de données. Dans cet article, je présente les principales étapes permettant de créer une API REST avec Node.js, Express et MongoDB.

Qu'est-ce qu'une API REST ?

Une API (Application Programming Interface) est une interface qui permet à plusieurs applications d'échanger des informations.

REST (Representational State Transfer) est une architecture qui utilise les méthodes HTTP classiques :

  • GET : récupérer des données
  • POST : créer une nouvelle ressource
  • PUT : modifier une ressource existante
  • DELETE : supprimer une ressource

Exemple pour une bibliothèque numérique :

  • GET /books récupère tous les livres
  • GET /books/1 récupère un livre précis
  • POST /books ajoute un nouveau livre
  • DELETE /books/1 supprime un livre

Préparation du projet

La première étape consiste à créer un nouveau projet Node.js.

mkdir api-rest-demo
cd api-rest-demo

npm init -y

Ensuite, nous installons les dépendances nécessaires :

npm install express mongoose cors dotenv
npm install --save-dev nodemon
  • Express : framework backend
  • Mongoose : connexion à MongoDB
  • CORS : autorisation des requêtes externes
  • Dotenv : gestion des variables d'environnement
  • Nodemon : redémarrage automatique du serveur

Création du serveur Express

Créons un fichier server.js.

const express = require("express");

const app = express();

app.use(express.json());

app.get("/", (req, res) => {
  res.send("API opérationnelle");
});

app.listen(3000, () => {
  console.log("Serveur lancé sur le port 3000");
});

Le serveur est maintenant capable de recevoir des requêtes HTTP.

Connexion à MongoDB

Pour stocker les données, nous utilisons MongoDB.

const mongoose = require("mongoose");

mongoose
  .connect(process.env.MONGO_URI)
  .then(() => console.log("MongoDB connecté"))
  .catch((err) => console.error(err));

Dans le fichier .env :

MONGO_URI=votre_url_mongodb

Cette configuration permet à l'application de communiquer avec la base de données.

Création d'un modèle de données

Dans MongoDB, les données sont généralement représentées à travers des modèles. 

const mongoose = require("mongoose");

const BookSchema = new mongoose.Schema({
  title: String,
  author: String,
  year: Number,
});

module.exports = mongoose.model("Book", BookSchema);

Ce modèle définit la structure des documents enregistrés dans la base de données.

Création des routes

Récupérer tous les livres

router.get("/", async (req, res) => {
  const books = await Book.find();
  res.json(books);
});

Ajouter un livre

router.post("/", async (req, res) => {
  const book = await Book.create(req.body);

  res.status(201).json(book);
});

Supprimer un livre

router.delete("/:id", async (req, res) => {
  await Book.findByIdAndDelete(req.params.id);

  res.json({
    message: "Livre supprimé",
  });
});

Gestion des erreurs

Une API professionnelle doit toujours gérer les erreurs.

try {
  const books = await Book.find();

  res.json(books);
} catch (error) {
  res.status(500).json({
    message: "Erreur serveur",
  });
}

Cette approche évite les plantages et permet de retourner des messages compréhensibles aux utilisateurs.

project/
│
├── controllers/
├── models/
├── routes/
├── middleware/
├── config/
├── .env
├── server.js

Cette organisation facilite l'évolution de l'application lorsque le nombre de fonctionnalités augmente.

Conclusion

Créer une API REST avec Node.js et Express est une excellente manière de comprendre le fonctionnement du backend moderne. Grâce à Express, la gestion des routes est simple et rapide, tandis que MongoDB permet de stocker efficacement les données.

Durant mon parcours en développement full stack, l'apprentissage des API REST m'a permis de mieux comprendre les échanges entre le frontend et le backend, ainsi que l'importance d'une architecture claire et maintenable.

La maîtrise des API constitue aujourd'hui une compétence essentielle pour tout développeur souhaitant construire des applications web modernes et évolutives.
